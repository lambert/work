Documentation.

Revision:20090505

Identification.

All documents are to be uniquely numbered.

Correspondence.

All correspondence documents to be stored/prepared in Microsoft Outlook Message (.msg) or plain ASCII text (.txt) format and to be uniquely numbered by the following numbering scheme: "work/xx_${unit_name}/documentation/yyyymmdd_${description}.ext"
Where:
  • xx - unit number
  • yyyy - year number
  • mm - month number
  • dd - day number
  • ext - file extension

Specifications.

All specification documents to be prepared in Hyper Text Markup Language format (.html), or plain ASCII text format (.txt), or Printable Document Format (.pdf), and to be uniquely numbered by the following numbering scheme: "work/xx_${unit_name}/documentation/xx_${description}.ext"
Where:
  • xx - unit number
  • ext - file extension

Schematics.

All schematics to be prepared with gEDA gschem.
File name convention: "work/xx_${unit_name}/electrical/gschem/xx.yyy.zz.aa.bb_${schematic_name}.sch"
Where:
  • xx - unit number
  • yyy - part number
  • zz - revision number
  • aa - document number
  • bb - page number

PCB layouts.

All printed circuit boards to be prepared with gPCB.
File name convention: "work/xx_${unit_name}/electrical/pcb/xx.yyy.zz.aa.bb_${pcb_name}.pcb"
Where:
  • xx - unit number
  • yyy - part number
  • zz - revision number
  • aa - document number
  • bb - page number

PCB Layer Configuration stack-up.

All pcb layer configuration stack-ups to be according to PCB Layer Configurations.

Footprints.

Naming of all footprint files and all embedded footprints to be in accordance with IPC standard IPC-7351A or any later version of the standard.
